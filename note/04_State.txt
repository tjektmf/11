1. State

    - 리액트 컴포넌트 내부의 변경되는 데이터
    - state 객체 내부에 정의된 데이터의 값이 변할 때마다 리액트가 자동으로 리렌더링 해준다
    - 리액트는 state 값이 변한 장소만 다시 그리는 방식으로 성능의 향상을 도모한다
    - 데이터 흐름에 반드시 필요한 값만 state 에 포함시켜야 한다 (무분별한 사용시 성능 저하를 유발함)


2. 클래스 컴포넌트

    - 요즘은 잘 쓰지 않는 방식의 컴포넌트
    - React.Component 클래스를 상속받아서 만들면 해당 클래스는 컴포넌트가 된다
    - state 객체는 클래스 컴포넌트에서만 사용해 볼 수 있다

3. 함수 컴포넌트

    -  props를 전달하면 생성한 엘리먼트를 반환하는 순수 함수
